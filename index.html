What is HTTP?

HTTP is the underlying protocol used by World Wide Web (WWW) and this protocol defines how messages are formatted and transmitted, and what actions Web servers and browsers should take in response to various commands.

For example, when you enter a URL in your browser, this actually sends an HTTP command to the Web server directing it to fetch and transmit the requested Web page.

How does HTTPS differ from HTTP?

HTTPS is a protocol for secure communications over HTTP protocol that is encrypted by TLS/SSL

HTTP on the other hand offers no traffic encryption thus sending data in its plain text form.

How are HTTP Requests formed?

  1. The client enters the URL in the browser.
  2. The browser looks up the IP for the domain name.
  3. The browser sends a HTTP request to the web server by establishing a TCP reliable connection with the Web Server and then transfer the request via this connection.

How are HTTP Responses formed?

  1. The web server waits for a HTTP request to be sent to them for processing.
  2. The server analyzes and interprets the HTTP request and executes the corresponding action based on the HTTP verb present on the request.
  3. After the server, processed and executed the action it sends a response back to the client.
